Scalable code

Writing scalable code means that if a new requirements come to you, in how much of a change, you can adapt to that requirement. The lesser the lines required, the more scalable the code is.
e.g. Say you are writing Node.js for Mac OSX and now you have to move to writing same for Linux/Debian based system. The code will be considered scalable on the basis of how many lines of code needs to be changed to adapt a Mac code to linux code.
